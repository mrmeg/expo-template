# Changelog

## [Unreleased]

- Added the authenticated billing API foundation: four Expo Server routes under `app/api/billing/` — `GET /summary`, `POST /checkout-session`, `POST /portal-session`, `POST /webhook` — all driven by a process-wide `BillingRegistry` so unconfigured deployments return a typed `503 billing-disabled` instead of throwing. Protected routes share `requireAuthenticatedUser` from new `app/api/_shared/auth.ts`, which extracts the bearer token, delegates to a pluggable `TokenVerifier` port, and fails closed with a structured 401 when no verifier is registered. Checkout maps `{ planId, interval }` onto a server-owned Stripe price id (400 `unknown-plan` with the catalog ids; 422 `configuration-missing` when the interval has no price) and surfaces resolver linking conflicts as 409 `billing-conflict`. The webhook route reads `request.text()` before any JSON parsing so the bytes Stripe signed match the bytes verified, dispatches through a `WebhookHandler` port, and short-circuits duplicate deliveries via a swappable `IdempotencyStore` (default in-memory LRU; multi-instance deployments inject a shared store via `setWebhookIdempotencyStore`). Checkout + Portal routes are now registered in `STRICT_LIMIT_PATHS` (10/min); the webhook is not, since Stripe retries can burst faster and its signature requirement already gates abuse. New `client/features/billing/api.ts` + `lib/problem.ts` fold the typed server errors into a `BillingProblem` discriminated union so UI code handles `billing-disabled`, `billing-conflict`, `no-customer`, `configuration-missing`, and `unknown-plan` without reading HTTP status codes. API reference now documents every route, body shape, and error code. Added 31 new tests (auth helper, idempotency store, all four routes with injected fakes, client problem mapping) — 238 total.
- Added the billing identity contract and normalized subscription state: new `shared/billing.ts` defines the `BillingSummary` shape, `freeBillingSummary`, `isEntitled`, `normalizeStripeSubscription`, and a default plan catalog. New `app/api/billing/_shared/` introduces `BillingAccountResolver` + `createStripeBillingAccountResolver` with deterministic customer-linking rules (metadata lookup → single-match email backfill → `CustomerConflictError` → create) and reads billing state without auto-creating customers. New `client/features/billing/` exposes the `useBillingSummary` React Query hook as the single entry point for subscription state — the profile tab now reads plan label, renewal date, and status from it instead of hard-coded "Free" / "Active" labels. Raw Stripe SDK types do not cross the client boundary. Added unit coverage for shared billing helpers and the resolver (17 new tests, 204 total).
- Defined the default Stripe subscriptions architecture: new `Agent/Docs/BILLING.md` names Stripe Checkout + Billing Portal in `hosted-external` mode as the template baseline, with webhook-authoritative server state, a normalized `BillingSummary` keyed to Cognito `sub`, a single `/billing/return` contract (web `https://…`, native `myapp://billing/return`), and explicit applicability guardrails (native PaymentSheet, App Store / Play Store in-app purchase, metered billing, tax, and team seats are out of scope for the baseline). ARCHITECTURE, API, APP_OVERVIEW, DOMAIN, and USER_FLOWS docs all reference the billing baseline.
- Wired the app shell to auth and onboarding: new `client/features/app/` module (`useAppStartup`, `AuthGate`, `OnboardingGate`, `isAuthEnabled`) owns startup sequencing and per-surface auth policy. Splash stays visible until fonts, i18n, onboarding, and (optionally) Amplify bootstrap have resolved. Onboarding is rendered inline by the root layout on first run — the main Stack never mounts during onboarding. Profile and Settings tabs wrap their exports in `<AuthGate>`, so signing out replaces those tabs with `AuthScreen` inline while unprotected surfaces stay browsable. The template remains fully explorable without Cognito env vars. Added unit coverage for `isAuthEnabled` and `AuthGate`.
- Fixed pre-existing Jest failures (react-test-renderer pin, expo-font / vector-icons / safe-area-context mocks, Progress accessibility role, Switch thumb tree assertion, InputOTP onComplete via simulated changeText).
- Fixed upload rate-limit alignment: the strict 10-requests-per-minute limiter now covers `/api/media/getUploadUrl` (the real presigned-upload route). Configuration moved to `server/rateLimits.js` with regression coverage so the path cannot drift back to the stale `/api/media/upload-url`.
- Fixed web media delete: shared CORS helper now advertises `DELETE` so browser preflights to `DELETE /api/media/delete?key=...` succeed from allowed origins. Added regression coverage in `app/api/_shared/__tests__/cors.test.ts`.
- Fixed false-positive client env warnings on web by validating `EXPO_PUBLIC_*` values with Expo-compatible direct property access.
- Disabled RN core `useNativeDriver` on web to remove repeated dev warnings from showcase and demo animations.
- Blurred focused web navigation elements before route changes to avoid `aria-hidden` warnings when hidden screens retained focus.
- Fixed the web accordion wrapper so uncontrolled accordions no longer flip into controlled mode on first toggle.
