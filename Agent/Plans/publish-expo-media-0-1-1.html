Publish Expo Media 0.1.1

Status: Blocked

Priority: High

Area: packages/media release workflow

Blocked by: npm trusted publishing setup or a usable NPM_TOKEN for @mrmeg/expo-media@0.1.1.

Goal

Publish the already committed @mrmeg/expo-media@0.1.1 package to npm without bumping to a workaround version, then verify npm reports 0.1.1 as latest.

Current Evidence

  • packages/media/package.json declares version 0.1.1.
  • A prior registry check reported npm latest as 0.1.0.
  • The publish workflow reached npm publish and failed with ENEEDAUTH after package gates, pack, and consumer smoke had already passed.

Plan

  1. Verify the package artifact locally or in CI: 
    bun run media:typecheck
bun run media:test
bun run media:build
bun run media:pack
bun run media:consumer-smoke
  2. Inspect the failed publish-media.yml run logs and classify the blocker as missing npm trusted publishing setup or missing/invalid NPM_TOKEN.
  3. Fix the credential path. Prefer npm trusted publishing for owner mrmeg, repository expo-template, workflow publish-media.yml. Use the token fallback only if that is the intended configured path.
  4. Rerun the publish workflow manually for version 0.1.1 and the committed ref.
  5. Verify the registry: 
    npm view @mrmeg/expo-media version dist-tags --json

Acceptance Criteria

  • npm view @mrmeg/expo-media version --json returns "0.1.1".
  • The successful GitHub Actions run is recorded in the final report.
  • Package validation gates pass before publishing.
  • The final report states whether trusted publishing or NPM_TOKEN was used.
  • No new version bump is committed unless 0.1.1 is already published.

Constraints

  • Do not publish from a local developer machine without explicit human approval.
  • Do not rerun the same failing publish job without first classifying the credential blocker.
  • Keep validation focused on packages/media.
  • If credentials cannot be configured by the agent, leave the plan blocked with the precise missing setup.
